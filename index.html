  • MS + W3C develop Pointer Events

Backend == Frontend

Of course not. But what a user sees of a Backend is a frontend.

Use what you have

  • Use native elements (e.g. forms)
  • Be progressive
  • Care about performance in the Back-Frontend
Instead of jQuery datepickers use a native with a polyfill. Instead of generating DOM via JS only inject data not the full DOM. No one wants to wait 3secs to load a page. Not even in a backend.
